# Contributing to Claude Control

Thanks for your interest in contributing! This project is actively maintained and contributions are welcome. To keep things productive for everyone, please follow these guidelines.

## Before You Code

**Open an issue first.** Before starting work on any non-trivial change, please open an issue to discuss your idea. This helps us:

- Align on whether the change fits the project's direction
- Avoid duplicate or conflicting work
- Discuss the best approach before investing time in implementation

This is especially important for:

- New features or significant behavior changes
- Architectural changes or refactors
- Large additions (new files, new dependencies)

Small fixes (typos, minor bug fixes, documentation improvements) can go straight to a PR.

## Pull Request Guidelines

- **Keep PRs focused and reasonably sized.** Large PRs that change many things at once are difficult to review and are likely to be closed. If your change is large, break it into smaller, incremental PRs that can each be reviewed independently.
- **One concern per PR.** Don't mix unrelated changes in a single PR.
- **Reference the issue** your PR addresses.
- **Describe what and why** in the PR description — what problem does this solve, and why this approach?

## What We'll Close

PRs that arrive without prior discussion and introduce large, sweeping changes will generally be closed. This isn't personal — it's about making sure the project stays maintainable and contributions are aligned with its direction. You're always welcome to open an issue to discuss the idea first.

## Getting Help

If you're unsure whether something is worth proposing, open an issue and ask. We're happy to discuss ideas before any code is written.
